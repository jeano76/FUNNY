# 마켓MBTI 수동 테스트 가이드

## 테스트 환경 설정

### 1. 로컬 서버 실행
```bash
# 포트 8001에서 HTTP 서버 실행
python3 -m http.server 8001

# 또는 Node.js 사용
npx http-server -p 8001
```

### 2. 브라우저에서 접속
```
http://localhost:8001
```

## 테스트 항목

### 1️⃣ 홈페이지 기본 기능

- [ ] **페이지 로드**: 메인 로고, 제목, CTA 버튼 모두 표시
- [ ] **언어 감지**: 한국 사용자면 한글로 표시 (자동 감지)
- [ ] **반응형 디자인**:
  - 모바일 (375px) 정상 표시
  - 태블릿 (768px) 정상 표시
  - 데스크톱 (1024px+) 정상 표시
- [ ] **네비게이션**: 상단 메뉴 클릭 시 해당 페이지로 이동
- [ ] **OG 태그**: 소셜 공유 시 썸네일 표시 (DevTools 확인)

**콘솔 오류 확인**: F12 → Console 탭에서 빨간색 오류 없는지 확인

### 2️⃣ 퀴즈 기능

#### 2-1. 퀴즈 진행
- [ ] **페이지 로드**: `/quiz.html` 정상 로드
- [ ] **문항 로드**: 20개 문항 모두 표시 (Q1 ~ Q20)
- [ ] **선택 가능**: 각 문항마다 4개 선택지 클릭 가능
- [ ] **진행률**: 진행률 표시 업데이트 (1/20 → 2/20...)
- [ ] **이전 버튼**: "이전" 버튼 클릭 시 이전 문항으로 돌아가기
- [ ] **완료 제출**: 모든 문항 선택 후 "제출" 버튼 클릭

#### 2-2. 결과 페이지 이동
- [ ] **결과 표시**: 제출 후 `/result.html?type=XXXX` 페이지로 이동
- [ ] **성격 유형**: 결과 페이지에서 동물 캐릭터와 성격 유형 표시
- [ ] **상세 정보**:
  - 성격 설명 (특징)
  - 강점 (Strengths)
  - 약점 (Weaknesses)
  - 호환 유형 (Compatible Types)
  - 유명인 사례 (Famous Examples)

### 3️⃣ 결과 페이지

#### 3-1. 데이터 표시
- [ ] **한글 표시**:
  - 성격 유형 이름 (예: "고독한 전략가")
  - 성격 설명 (한국어)
  - 한국인 비율 (예: "2.1%")
  - 희귀도 표시
- [ ] **i18n 번역**: 모든 텍스트가 의도한 언어로 표시

#### 3-2. 공유 기능
- [ ] **공유 버튼**: 4개 공유 옵션 표시
  - 카카오톡 (Kakao Talk)
  - 트위터/X
  - Facebook
  - 링크 복사
- [ ] **카카오톡 공유**: "카카오톡으로 공유" 클릭 → 카카오톡 메시지 작성 창 열림
  - 이미지 포함
  - 성격 유형 정보 포함
- [ ] **트위터 공유**: 트위터 공유 대화상자 열림
- [ ] **링크 복사**: 클립보드에 복사되고 "복사됨" 메시지 표시

#### 3-3. 추가 기능
- [ ] **재시험**: "다시 시험하기" 버튼 → 퀴즈 페이지로 이동
- [ ] **홈**: "홈으로" 버튼 → 메인 페이지로 이동
- [ ] **호환성 비교**: "호환성 보기" 버튼 → 비교 페이지로 이동
- [ ] **이미지 저장**: "이미지로 저장" 또는 "카드 저장" 버튼 → 이미지 다운로드
- [ ] **직업 추천**: "직업 추천" 버튼 → 직업 추천 페이지로 이동

#### 3-4. localStorage 테스트
- [ ] **데이터 저장**: 결과 페이지 방문 후
  - DevTools → Application → Local Storage → localhost:8001
  - `mbti_result` 또는 유사 키 확인
- [ ] **다시 방문**: 새 탭에서 같은 URL 방문 → 이전 결과 유지
- [ ] **히스토리**: localStorage에 이전 결과 기록이 있는지 확인

### 4️⃣ 다국어 지원

#### 4-1. 자동 언어 감지
- [ ] **브라우저 설정**: 브라우저의 기본 언어를 변경
  - Chrome: Settings → Languages
  - 영어로 변경 후 `http://localhost:8001` 방문
  - 자동으로 영문 페이지로 이동하는지 확인

#### 4-2. 수동 언어 선택
- [ ] **언어 드롭다운**: 상단 네비게이션의 언어 선택기
- [ ] **지원 언어**: 다음 언어 선택 가능한지 확인
  - 한국어 (Korean)
  - English
  - 日本語 (Japanese)
  - 中文 (Chinese)
  - Español (Spanish)
  - Deutsch (German)
  - Français (French)
  - Русский (Russian)
  - Português (Portuguese)
  - Bahasa Indonesia (Indonesian)
  - हिन्दी (Hindi)
  - Tiếng Việt (Vietnamese)
  - ภาษาไทย (Thai)
  - Türkçe (Turkish)
  - Italiano (Italian)
  - Nederlands (Dutch)
  - العربية (Arabic)
  - Монгол (Mongolian)
  - Lingua Latina (Latin)

#### 4-3. 각 언어별 검증
각 언어를 선택했을 때:
- [ ] HTML의 `lang` 속성이 올바르게 설정되어 있는지 확인 (DevTools 확인)
- [ ] 페이지 콘텐츠가 해당 언어로 번역되어 있는지 확인
- [ ] 네비게이션 메뉴가 해당 언어로 표시되는지 확인
- [ ] 푸터가 해당 언어로 표시되는지 확인

예:
```
Korean: ✅
  - nav: "홈", "테스트", "비교", "직업", "언어"
  - footer: "개인정보 보호정책", "도움말", ...

English: ✅
  - nav: "Home", "Test", "Compare", "Careers", "Language"
  - footer: "Privacy Policy", "Help", ...
```

## 테스트 환경별 팁

### Chrome DevTools 활용
```
1. F12 또는 우클릭 → 검사
2. Device Toolbar: Ctrl+Shift+M (또는 Cmd+Shift+M)
3. Console: Ctrl+Shift+J (또는 Cmd+Option+J)
4. Network: Ctrl+Shift+E (또는 Cmd+Option+E)
5. Application: Ctrl+Shift+I → Application 탭
```

### localStorage 확인
```
DevTools → Application → Storage → Local Storage → http://localhost:8001
```

### 성능 측정
```
DevTools → Network → Network throttling: "Slow 3G" 선택
페이지 새로고침 → 로드 시간 확인
```

### 메타 태그 확인
```
DevTools → Elements → <head> 섹션 확인
또는 페이지 소스 보기 (Ctrl+U 또는 Cmd+U)
```
